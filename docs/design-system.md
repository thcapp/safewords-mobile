# Safewords Mobile — Design System Reference

> **Single source of truth** for designers and mobile engineers. Covers the three design themes (Ink, Signal, Paper), the A11Y / Plain-mode palette, typography, components, motion, and a per-token mapping from the design handoff to the iOS (Swift/SwiftUI) and Android (Kotlin/Compose) implementations.

---

## 1. Design Origin

The Safewords mobile visual language was generated by **Claude Design** and delivered to the team as an HTML/CSS/JS prototype bundle (the "design handoff bundle"). The bundle lives outside the repo and contains the following canonical assets:

```
safewords-app/
├── project/
│   ├── Safewords App.html      ← canonical rendered prototype (the spec)
│   ├── components/
│   │   ├── theme.jsx           ← token definitions (Ink, Signal, Paper, A11Y)
│   │   ├── primitives.jsx      ← CountdownRing, TabBar, GroupDot, SectionLabel, BigButton
│   │   └── screens/
│   │       ├── plain-mode.jsx  ← A11Y / Plain-mode screens
│   │       └── …               ← Home, Groups, Verify, Settings
│   └── …
└── chats/…                     ← design-rationale transcripts
```

**When the design bundle and the native code disagree, the bundle wins** — it is the spec. The native implementations below are ports and may lag the bundle. Update this document whenever the bundle is revised.

**Canonical source file**: `Safewords App.html`. All tokens, component geometry, and motion curves in this document were extracted from it and verified against the implementations.

Design origin for shipping tokens:

## 2. Themes Overview

Three design themes were specified in the handoff. Only **Ink** (the dark default) is implemented in native code today. Signal and Paper are documented for future work.

| Theme | Mood | Status |
|---|---|---|
| **Ink** | Editorial, dark-primary, ember accent. The default. | Implemented (iOS + Android) |
| **Signal** | Deep slate + cyan-sodium. Technical, cool, "status board" feel. | Specced, not implemented |
| **Paper** | Warm amber, light-background editorial. Daytime/print feel. | Specced, not implemented |
| **A11Y (Plain)** | High-contrast amber-on-navy for elderly + children. Runs alongside Ink. | Implemented (iOS + Android) |

The A11Y palette is orthogonal — it is not one of the three brand themes; it is an accessibility mode users opt into. It must meet WCAG AAA contrast targets.

## 3. Ink Theme — Token Table

Ink is editorial and near-monochrome. The single warm ember accent (`#E8553A`) carries all state — live indicators, progress arcs, primary actions. Greens and ambers are reserved for ok/warn semantics.

```
┌─────────────────────────────────────────────────────────┐
│  bg #0B0B0C                                             │
│   ┌───────────────────────────────────────────────────┐ │
│   │ bgElev #151517                                    │ │
│   │   ┌───────────────────────────────────────────┐   │ │
│   │   │ bgInset #1C1C1F    rule F5F2EC @ 8%       │   │ │
│   │   │                                           │   │ │
│   │   │     fg #F5F2EC         "Breezy Anchor"    │   │ │
│   │   │     fgMuted 55%        "rotates in 4m"    │   │ │
│   │   │     fgFaint  32%       "SEQ · 0142"       │   │ │
│   │   │                                           │   │ │
│   │   │     ● accent #E8553A   (LIVE, arc, CTA)   │   │ │
│   │   └───────────────────────────────────────────┘   │ │
│   └───────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────┘
```

### Token → Implementation

| Token | Design value | iOS (`Ink.*`) | Android (`Ink.*`) |
|---|---|---|---|
| `bg` | `#0B0B0C` | `Color(hex: "#0B0B0C")` | `Color(0xFF0B0B0C)` |
| `bgElev` | `#151517` | `Color(hex: "#151517")` | `Color(0xFF151517)` |
| `bgInset` | `#1C1C1F` | `Color(hex: "#1C1C1F")` | `Color(0xFF1C1C1F)` |
| `fg` | `#F5F2EC` | `Color(hex: "#F5F2EC")` | `Color(0xFFF5F2EC)` |
| `fgMuted` | `fg @ 55%` | `Color(hex: "#F5F2EC").opacity(0.55)` | `Color(0xFFF5F2EC).copy(alpha = 0.55f)` |
| `fgFaint` | `fg @ 32%` | `Color(hex: "#F5F2EC").opacity(0.32)` | `Color(0xFFF5F2EC).copy(alpha = 0.32f)` |
| `rule` | `fg @ 8%` | `Color(hex: "#F5F2EC").opacity(0.08)` | `Color(0xFFF5F2EC).copy(alpha = 0.08f)` |
| `accent` | `#E8553A` | `Color(hex: "#E8553A")` | `Color(0xFFE8553A)` |
| `accentInk` | `#0B0B0C` | `Color(hex: "#0B0B0C")` | `Color(0xFF0B0B0C)` |
| `ok` | `#9DBF8A` | `Color(hex: "#9DBF8A")` | `Color(0xFF9DBF8A)` |
| `warn` | `#E8A13A` | `Color(hex: "#E8A13A")` | `Color(0xFFE8A13A)` |
| `tickFill` | `accent @ 18%` | `Color(hex: "#E8553A").opacity(0.18)` | `Color(0xFFE8553A).copy(alpha = 0.18f)` |

**Rules**:
- Only one accent. Do not introduce a second hue for state.
- Never use pure `#FFF`/`#000` in Ink — `fg` is warm-cream `#F5F2EC` and `bg` is near-black `#0B0B0C`.
- `rule` (dividers) is always derived from `fg` at `8%` alpha — never a standalone color.
- `tickFill` at `18%` alpha is used for the countdown ring's elapsed-tick fill halo.

## 4. Other Themes — Signal & Paper (Specced, Not Implemented)

These are described in the handoff but have no native token implementation yet. When shipping them, mirror the Ink token structure and add a theme-switcher surface.

### Signal — deep slate + cyan-sodium

Cool, technical, "status board" aesthetic. Background in slate/indigo, accent in a sodium-cyan that reads like an LED. Intended as an alternate for power users and night-shift use.

Token shape (for future work): same keys as Ink (`bg`, `bgElev`, `bgInset`, `fg`, `fgMuted`, `fgFaint`, `rule`, `accent`, `accentInk`, `ok`, `warn`, `tickFill`).

### Paper — warm amber

Editorial light theme with paper-cream background and a warm amber accent. Print/daytime feel; higher contrast than Ink for bright-environment reading.

Token shape: same keys as Ink. Inverted — `bg` is warm cream, `fg` is near-black, `rule` derives from `fg`.

Both themes require running the full token table and QA pass against the handoff bundle before shipping.

## 5. A11Y (Plain-Mode) Palette

Plain mode is a **WCAG AAA high-contrast mode** for elderly users and children. Amber-on-navy at very large type. Minimum 72dp hit targets. No jargon in copy.

```
┌──────────────────────────────────────────────┐
│  bg #0B1220  (deep navy, not pure black)     │
│                                              │
│   ┌────────────────────────────────────────┐ │
│   │ bgElev #18243C                         │ │
│   │                                        │ │
│   │    fg #FFFFFF        (21.0 : 1)        │ │
│   │    fgMuted #CBD5E1   (11.6 : 1) AAA    │ │
│   │    fgFaint #94A3B8   ( 6.4 : 1) AA     │ │
│   │                                        │ │
│   │    ● accent #FFD23F  (13.8 : 1) AAA    │ │
│   │    ✓ ok     #4ADE80  ( 9.3 : 1) AAA    │ │
│   │    ✗ danger #FF6B6B  (reserved)        │ │
│   └────────────────────────────────────────┘ │
└──────────────────────────────────────────────┘
```

### Token → Implementation

| Token | Design value | iOS (`A11Y.*`) | Android (`A11y.*`) |
|---|---|---|---|
| `bg` | `#0B1220` | `Color(hex: "#0B1220")` | `Color(0xFF0B1220)` |
| `bgElev` | `#18243C` | `Color(hex: "#18243C")` | `Color(0xFF18243C)` |
| `bgInset` | `#24354F` | `Color(hex: "#24354F")` | `Color(0xFF24354F)` |
| `fg` | `#FFFFFF` | `Color.white` | `Color.White` |
| `fgMuted` | `#CBD5E1` (11.6:1 on bg) | `Color(hex: "#CBD5E1")` | `Color(0xFFCBD5E1)` |
| `fgFaint` | `#94A3B8` (6.4:1 on bg) | `Color(hex: "#94A3B8")` | `Color(0xFF94A3B8)` |
| `rule` | `#FFF @ 22%` | `Color.white.opacity(0.22)` | `Color.White.copy(alpha = 0.22f)` |
| `accent` | `#FFD23F` (13.8:1 on bg) | `Color(hex: "#FFD23F")` | `Color(0xFFFFD23F)` |
| `accentInk` | `#0B1220` | `Color(hex: "#0B1220")` | `Color(0xFF0B1220)` |
| `ok` | `#4ADE80` (9.3:1) | `Color(hex: "#4ADE80")` | `Color(0xFF4ADE80)` |
| `danger` | `#FF6B6B` | `Color(hex: "#FF6B6B")` | `Color(0xFFFF6B6B)` |
| `tickFill` | `accent @ 22%` | `Color(hex: "#FFD23F").opacity(0.22)` | `Color(0xFFFFD23F).copy(alpha = 0.22f)` |

**A11Y contrast targets** (all vs. bg `#0B1220`):
- `fg` → 21.0 : 1 (AAA)
- `fgMuted` → 11.6 : 1 (AAA)
- `fgFaint` → 6.4 : 1 (AA large text only; do not use for body)
- `accent` → 13.8 : 1 (AAA)
- `ok` → 9.3 : 1 (AAA)

**Sizing rules in A11Y**: body minimum 20px, hit targets minimum 72dp, display 28–48px bold. See §6.

## 6. Typography

Ink uses a three-family system: a serif display for brand moments, a sans for body UI, a mono for technical meta. A11Y swaps display+body for Atkinson Hyperlegible.

### 6.1 Ink typography

| Role | Design family | Fallback | iOS | Android |
|---|---|---|---|---|
| Display | **Fraunces** (serif) | system serif | `Fonts.display(size, weight)` — custom if registered else `.system(size:, design: .serif)` | `displayStyle(size, tracking)` — system serif via default Material typography (custom font not yet bundled) |
| Body | **Inter** (sans) | system sans | `Fonts.body(size, weight)` — `.system(size:, design: .default)` | Material `Typography` bodyLarge/Medium/Small (system default) |
| Mono | **JetBrains Mono** | system mono | `Fonts.mono(size, weight)` — `.system(size:, design: .monospaced)` | `monoStyle(size, tracking)` + `FontFamily.Monospace` |

Neither platform currently registers the custom `.otf`/`.ttf` files. The resolver falls back to the platform's nearest system family. **When you bundle the webfonts, update `Fonts.display` (iOS) and the `Typography` object (Android) to resolve the custom family first.**

### 6.2 Size scale (extracted from `Safewords App.html`)

| Use | Size | Weight | Tracking | Family |
|---|---|---|---|---|
| Hero word lines (Home) | 46 | Regular | −1.5 | Display (Fraunces) |
| Page display (onboarding) | 28 | Regular | −0.5 | Display |
| Section label / eyebrow | 11 | Medium | +1.4, uppercase | Body (Inter) |
| Body default | 15 | Regular | 0 | Body |
| Body small | 13 | Regular | 0 | Body |
| Body tiny | 12 | Regular | 0 | Body |
| Tab bar label | 10.5 | Medium | +0.2 | Body |
| Countdown digits | 28 | Regular | +2 | Mono |
| Meta mono (`SEQ · 0142`) | 11 | Regular | +1.5 | Mono |

### 6.3 A11Y typography

- **Family**: Atkinson Hyperlegible for both display and body. Falls back to system sans if unregistered.
- **Display**: 36–48pt bold for the live word; never below 28pt.
- **Body**: 20pt minimum. 22pt bold for BigButton labels.
- **No mono** — mono type is hard to read at a11y sizes.

iOS resolver: `A11yFonts.body(size, weight)` / `A11yFonts.display(size, weight)` in `PlainModeViews.swift`.
Android resolver: use Material `Typography` with the A11y font family once bundled; until then, system default with Bold weight.

## 7. Avatar Palette (DotPalette)

Five brand-aligned hues used for group/member avatar dots. Chosen for cross-theme contrast. Indexed deterministically from stable group id so the same family always gets the same color on every device.

| Name | Hex | Swatch | iOS | Android |
|---|---|---|---|---|
| red | `#E8553A` | ember (matches Ink accent) | `DotPalette.red` | `DotPalette[0]` |
| blue | `#6E94E7` | periwinkle | `DotPalette.blue` | `DotPalette[1]` |
| green | `#9DBF8A` | sage (matches Ink ok) | `DotPalette.green` | `DotPalette[2]` |
| orange | `#E89B5E` | amber-orange | `DotPalette.orange` | `DotPalette[3]` |
| purple | `#B47AE8` | lilac | `DotPalette.purple` | `DotPalette[4]` |

**Assignment rule**: `palette[((index % 5) + 5) % 5]` using the group's stable hash. iOS: `DotPalette.forIndex(i)`; Android: `DotPalette[((i % 5) + 5) % 5]`.

**On-color for initials**: always `#FFFFFF`. This is legible on all five dots.

## 8. Components

### 8.1 CountdownRing

The central identity of the app. A dashed-dial ring around the current safeword showing time until rotation.

```
         tick 0 (12 o'clock) — big tick
              │
         ·───○───·       ← knob (accent, 10dp dot + 16dp stroke halo)
       ·         ·
     ·             ·     ← every 5th tick is 1dp wide, longer;
    ·    Breezy    ·       others are 0.6dp, shorter
    ·    Anchor    ·
    ·     47       ·     ← hero content (inside the ring)
     ·             ·
       ·         ·
         · · · · ·       ← ember arc sweeps clockwise from 12
```

**Geometry**:
- 60 ticks total, evenly distributed around 360°.
- Every 5th tick (i % 5 == 0) is "big": longer inward (6dp) and outward (2dp), stroke width 1dp.
- Normal ticks: inward 3dp, outward 0dp, stroke width 0.6dp, round cap.
- Elapsed ticks use `fgFaint @ 90%`; unelapsed use `fgFaint @ 25%`.
- Progress arc: full circle trimmed by `progress` (0…1), starts at −90° (12 o'clock), stroke `accent` at 1.5dp, round cap.
- Knob: 10dp `accent` filled circle + 16dp `accent @ 25%` stroke halo on the leading arc edge.
- Canonical ring frame: **340 × 340 dp**; inner content padding 24dp horizontal.

**Implementations**:
- iOS: `Views/Components/CountdownRing.swift` — `GeometryReader` + `ForEach(0..<60)` drawing `Path`s; progress arc is a trimmed `Circle` rotated −90°.
- Android: `ui/components/CountdownRing.kt` — `Canvas` with `drawLine` loop and `drawArc` at startAngle −90°. Animates via `animateFloatAsState(tween(500, LinearEasing))`.

### 8.2 Floating Tab Bar

Pill-shaped, floating above content with drop shadow. Four tabs in Ink; three tabs in A11Y.

**Geometry**:
- Outer pill radius: **28dp** (Ink) / 26dp (A11Y).
- Inner active-tab chip radius: **22dp** (Ink) / 20dp (A11Y).
- Padding: 6dp inside the pill.
- Shadow: `black @ 35%` (Ink) / `black @ 45%` (A11Y), radius 40, y 10.
- Border: 0.5dp `rule` (Ink) / 2dp `rule` (A11Y).
- Horizontal margin: 12dp (Ink) / 10dp (A11Y). Bottom margin: 26dp (Ink) / 34dp (A11Y).

**Active state**: active tab gets `bgInset` chip behind it (Ink) or full `accent` fill (A11Y).

**iOS**: `CustomTabBar.swift` (Ink) / `A11yTabBar` in `PlainModeViews.swift` (A11Y).
**Android**: `ui/components/DesignComponents.kt` surfaces the pill wrapper; screens compose tabs inline.

### 8.3 GroupDot

Solid circle avatar with a single capped initial.

- Default size: **36dp**. Also used at 24dp in the top-bar group pill and 48dp in list rows.
- Fill: one of the 5 `DotPalette` colors.
- Initial: first char, uppercase, white, `SemiBold`, font size = `size × 0.42`.
- No border.

**iOS**: `GroupDot.swift`. **Android**: `GroupDot()` in `DesignComponents.kt`.

### 8.4 SectionLabel

An uppercase eyebrow label used above cards and list sections.

- Size: **11**, weight `Medium`.
- Tracking: **+1.4**.
- Always uppercase.
- Color: `fgMuted` by default; `accent` when used with a LIVE dot (e.g. `● LIVE · JOHNSON FAMILY`).

**iOS**: `SectionLabel.swift`. **Android**: `SectionLabel` in `DesignComponents.kt`.

### 8.5 Glass pill button (group selector, bell)

Top-bar control style in Ink.

- Background: `bgElev`.
- Border: `rule` at 0.5dp.
- Shape: `Capsule` (pill) for group selector; `Circle` at 38dp for the bell.
- Padding for pill: leading 7, trailing 12, vertical 7.
- Typography inside: 13 Medium (body).

### 8.6 BigButton (A11Y only)

Large, icon-prefixed CTA for Plain mode. Two variants: `primary` (accent fill) and `ghost` (bgElev fill, rule border).

- Minimum height: **72dp**.
- Radius: 18dp.
- Padding: 22dp horizontal, 18dp vertical.
- Label: A11y body at 22pt Bold, tracking −0.2.
- Icon: 22pt Bold, leading.
- Primary fg: `A11Y.accentInk`; ghost fg: `A11Y.fg`.

**iOS**: `BigButton` in `PlainModeViews.swift`. **Android**: ship alongside other A11Y primitives (parity task).

### 8.7 ElevatedCard (Android-specific wrapper)

Reusable ghost card. Corner radius **20dp**, `bgElev` fill, `rule` border at 0.5dp. Use for list rows, info blocks, verify panels. iOS uses ad-hoc `RoundedRectangle` per view with the same tokens.

## 9. Reveal Styles (Home hero)

The word can be revealed three ways, user-selectable in Settings.

| Style | Behavior |
|---|---|
| `always` (default) | Word is always visible. |
| `holdReveal` | Word is blurred 14px; press-and-hold to reveal. Shows "Hold to reveal" hint below the ring while un-held. |
| `ambient` | Word displayed with lower contrast / fading-in animation for peripheral glanceability (specced; implementation simplified today). |

**Blur**: `14px` radius on the word `VStack`.
**Animation**: `easeInOut` 180ms on the held state.
**Gesture**: `DragGesture(minimumDistance: 0)` — press begins reveal, release ends it. iOS: `HomeView.holdGesture`.

The user preference is stored as `revealStyle` (string) in user defaults / shared prefs.

## 12. Motion

All motion is quiet and deliberate. No bounce, no overshoot.

| Event | Duration | Easing | Notes |
|---|---|---|---|
| Blur reveal (hold-to-reveal) | **180 ms** | `easeOut` | Applied to the word VStack's blur radius. |
| Countdown arc tween | **500 ms** | `linearEasing` | `animateFloatAsState` (Android) / SwiftUI implicit animation on `progress` (iOS). |
| Listening pulse (Verify) | **2400 ms** repeating | `easeOut` | Concentric-ring opacity + scale. One-shot per cycle, then restart. |
| Tab switch chip | 180 ms | `easeInOut` | Background chip crossfade. |
| Ghost card press | 120 ms | `easeOut` | Optional subtle scale to 0.98. |

**Never**: spring physics on the word hero, haptic-paired scaling on the countdown, or fast (<150ms) transitions on anything cryptographic.

## 14. Do / Don't

**Do**
- Derive new colors by adjusting alpha on an existing foreground or accent (never introduce new hex values).
- Use mono only for technical meta (`SEQ`, countdown digits, hex seeds).
- Keep a single accent per theme. LIVE indicators and CTAs share the same hue.
- In A11Y, always verify contrast against bg (`#0B1220`) and target AAA for body text.

**Don't**
- Don't mix Ink and A11Y tokens on the same screen — the user is in one mode at a time.
- Don't tint icons with arbitrary colors; use `fg`, `fgMuted`, or `accent`.
- Don't compose your own countdown ring — use the primitive. The 60-tick / every-5th / knob spec is load-bearing for brand recognition.
- Don't add custom radii. If you need a radius not in §11, raise it with design.
- Don't use pure black or pure white in Ink.
